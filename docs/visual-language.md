# EchoCanvas 视觉语言设计

> 设计原则: 每种思维类型一眼可辨，颜色+形状双重编码，暗色主题优先

## 5 种思维节点

### 1. Analysis (分析) - 圆角矩形 + 蓝色
- **形状**: 圆角矩形 (最基础，出现频率最高)
- **颜色**: `#3B82F6` (蓝色系) → 冷静、理性
- **图标**: 🔍 放大镜 (Lucide: `Search`)
- **用途**: 观察现象、收集信息、分析数据
- **示例**: "看到报错信息...", "读取文件发现...", "日志显示..."

### 2. Hypothesis (假设) - 菱形 + 紫色
- **形状**: 菱形/钻石 (表示"可能性"，有方向感)
- **颜色**: `#8B5CF6` (紫色系) → 想象、推测
- **图标**: 💡 灯泡 (Lucide: `Lightbulb`)
- **用途**: 提出猜想、推测原因、设想方案
- **示例**: "可能是因为...", "如果...", "也许..."

### 3. Verification (验证) - 八边形 + 绿色
- **形状**: 八边形 (类似"检查站")
- **颜色**: `#10B981` (绿色系) → 验证、通过
- **图标**: ✓ 勾选 (Lucide: `CheckCircle`)
- **用途**: 测试假设、运行代码、检查结果
- **示例**: "运行测试...", "build 通过", "验证结果..."

### 4. Decision (决策) - 六边形 + 琥珀色
- **形状**: 六边形 (蜂巢=枢纽，表示关键决策点)
- **颜色**: `#F59E0B` (琥珀色) → 重要、需要注意
- **图标**: ⚡ 闪电 (Lucide: `Zap`)
- **用途**: 做出选择、确定方向、权衡利弊
- **示例**: "决定采用方案A", "选择...", "最终..."
- **特殊**: 从此节点可能分出多条路径(被选中的和被放弃的)

### 5. Action (行动) - 圆形 + 红橙色
- **形状**: 圆形 (完成感，句号)
- **颜色**: `#EF4444` (红橙色) → 执行、行动
- **图标**: ▶ 播放 (Lucide: `Play`)
- **用途**: 执行具体操作、写代码、提交、部署
- **示例**: "修改了...", "提交 commit", "部署完成"

## 4 种连线类型

| 类型 | 样式 | 颜色 | 含义 |
|------|------|------|------|
| `leads_to` | 实线 + 箭头 | `#64748B` (灰色) | 逻辑推导 (A→B) |
| `supports` | 实线 + 圆点 | `#10B981` (绿色) | 证据支持 |
| `contradicts` | 虚线 + 红色 | `#EF4444` (红色) | 矛盾/否定 |
| `branches` | 虚线 + 分叉 | `#F59E0B` (琥珀) | 方案分支 |

## 确信度可视化

- 节点边框粗细表示确信度 (0-1)
  - `< 0.3`: 细虚线边框 + 半透明
  - `0.3-0.7`: 正常边框
  - `> 0.7`: 粗边框 + 微发光效果


## 布局规则

- 思维流从左到右排列 (时间方向)
- dagre 布局: `rankdir: 'LR'`, `nodesep: 60`, `ranksep: 120`
- 被放弃的分支向下偏移，降低视觉权重 (透明度 0.4)
